# Knucklebones Website Brief

This document is for a separate Codex session updating the Iron Quill website to feature the upcoming game **Knucklebones**.

It is intended to give that session enough context to write strong website copy, choose the right calls to action, and link to the correct destinations without having to inspect the game project deeply.

## What Knucklebones Is

**Knucklebones** is an atmospheric, systems-driven dark fantasy board game / dice strategy game built around:

- staged dice rolling
- tactical route choice across a physical parchment map
- node events, traps, and special encounters
- combat encounters using dice assignment and resolution
- persistent run progression through dice, inventory cards, wounds, curses, and player-state changes

The game is not framed like a bright abstract tabletop toy. It is tactile, moody, and ceremonial.

The presentation language is:

- candlelight
- parchment maps
- wood, bone, heads, carved wheels
- dark folklore / mythic roadside ritual
- physical objects arranged on a table rather than flat menu-first UI

The project already leans hard into authored atmosphere. It should not be marketed as generic “indie roguelike deckbuilder” material. The website should present it as a distinctive physical-feeling dark strategy game with dice, board traversal, and ritual tone.

## High-Level Genre Positioning

Use language in the following territory:

- dark fantasy dice strategy
- board-game-inspired roguelite
- ritual tabletop journey
- tactical run-based progression

Do **not** reduce it to:

- “just a dice game”
- “just a roguelike”
- “just a deckbuilder”

Those are incomplete and flatten what is distinctive about the project.

## Core Player Fantasy

The player is not simply clicking through menus. The intended feel is:

- choosing a character/run identity
- physically committing dice
- moving across a dangerous parchment route
- surviving traps, wounds, curses, and enemies
- shaping the run through risk, inventory, and route choices

The game’s onboarding and death sequences have both been receiving bespoke treatment in the project specifically because the experience is meant to feel authored, slow-burning, and ritualistic rather than disposable.

That tone should be reflected on the website.

## Current Project Shape

From the current codebase and project documentation, the game includes:

- a main menu with direct Steam / social / studio links
- a new-run `Select Player` onboarding phase
- map generation and traversal
- node-based progression
- trap and event cards
- combat encounters
- dice rolling, staging, and assignment systems
- player health / shield wheels
- inventory cards including pip-adjustment systems
- death sequence / run loss flow
- save / continue support

This means the website can truthfully present the game as an actively built, feature-rich upcoming title rather than a vague concept.

## The Most Important Website Goal

Make visitors want to:

1. understand the game quickly
2. feel its atmosphere immediately
3. click through to Steam and wishlist it

The website should not read like a design document.
It should not over-explain the rules.
It should communicate:

- what the game feels like
- why it is distinctive
- why someone who likes dark tactical systems should care

## Recommended Messaging Direction

Good messaging themes:

- roll the bones
- travel a cursed road
- choose your route carefully
- survive traps, curses, and hostile encounters
- shape your run through dice and hard decisions
- ritual, flame, parchment, fate, bone

Avoid messaging that sounds:

- corporate
- overly clean
- generic roguelike boilerplate
- too whimsical
- too “cozy tabletop”

## Suggested Short Description

Use something in this range:

> Knucklebones is a dark fantasy dice strategy game where you travel a hand-drawn parchment map, survive traps and encounters, and shape each run through tactical rolls, route choices, and grim tradeoffs.

Alternative:

> Knucklebones is a board-game-inspired dark fantasy roguelite built around staged dice, dangerous routes, strange encounters, and tactile ritual atmosphere.

## Suggested Medium Description

Use something like:

> In Knucklebones, every run begins at the table: candlelight, parchment, carved wheels, and a road laid out before you. Roll and stage your dice, choose your path across a dangerous map, survive traps and combat encounters, and manage the wounds, curses, and rewards that shape your journey. Knucklebones blends tactile board-game presentation with dark fantasy strategy and run-based progression.

## Key Selling Points

These are safe to surface on the website:

- Tactile dark fantasy presentation grounded in physical objects and candlelit atmosphere
- Dice-driven strategy rather than passive random resolution
- Board / route traversal with meaningful node choice and map readability
- Encounters, traps, combat, and status effects that alter the run
- Persistent run-shaping systems including wounds, curses, inventory, and dice changes

## Potential Feature Bullets

- Stage and commit dice with intent
- Traverse a dangerous parchment map
- Face traps, events, and hostile encounters
- Manage wounds, curses, and hard tradeoffs
- Shape each run through route choice and evolving dice

## Visual Direction For The Website

The website should harmonize with the game’s actual tone:

- aged parchment
- deep blacks
- warm candlelight
- carved / inked / ritual textures
- restrained typography with conviction
- physicality over flat SaaS-like layout

Do not market it with:

- clean neon gradients
- bright toy-like colors
- generic fantasy wallpaper
- overdesigned modern startup UI language

If the site includes screenshots or hero treatment, the game should read as:

- dark
- authored
- tactile
- legible
- serious in tone

The website should push wishlist intent first.

## Notes For Website Copywriting

- Lead with atmosphere and identity first, mechanics second.
- Mention dice, map traversal, and encounters early.
- Keep “wishlist on Steam” visible and repeated where appropriate.
- Avoid jargon-heavy system explanation on the landing surface.
- If there is a deeper game section lower on the page, explain:
  - route choice
  - dice staging
  - encounters / traps / combat
  - wounds / curses / evolving run state

## Notes For Website Layout

If the website is being updated substantially, a strong structure would be:

1. Hero
   - game title
   - one-sentence hook
   - wishlist CTA
2. Atmosphere / identity section
   - what Knucklebones feels like
3. How it plays
   - dice, route choice, encounters
4. Visual section
   - screenshots / art / motion
5. CTA close
   - wishlist again

## What Not To Claim

Do not claim anything that is not clearly supported by the project today, such as:

- release date
- multiplayer
- co-op
- controller support
- platform support beyond what the studio has explicitly announced
- narrative features that are not actually established

## Bottom-Line Positioning

If another Codex session needs a single guiding sentence, use this:

**Present Knucklebones as a dark, tactile, board-game-inspired dice strategy game with strong ritual atmosphere, meaningful route choice, and run-shaping encounters, then drive the visitor toward wishlisting it on Steam.**
